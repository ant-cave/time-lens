# Vue3 路由与响应式侧边栏开发记录

## 日期
2025-03-28

## 概述
本次开发为 Time Lens 项目添加了 Vue Router 路由系统，以及一个支持响应式布局的可收缩侧边栏。

---

## 一、Vue Router 基础配置

### 1.1 安装路由
```bash
pnpm add vue-router@4
```

### 1.2 创建路由配置文件
**文件**: `src/router/index.js`

```javascript
import { createRouter, createWebHashHistory } from 'vue-router'
import Today from '../views/Today.vue'
import Calendar from '../views/Calendar.vue'
import Analysis from '../views/Analysis.vue'
import Settings from '../views/Settings.vue'
import About from '../views/About.vue'

// 路由配置
const routes = [
  { path: '/', redirect: '/today' },  // 重定向到今日页面
  { path: '/today', component: Today },
  { path: '/calendar', component: Calendar },
  { path: '/analysis', component: Analysis },
  { path: '/settings', component: Settings },
  { path: '/about', component: About },
]

const router = createRouter({
  history: createWebHashHistory(),  // 使用 hash 模式
  routes,
})

export default router
```

**知识点**:
- `createRouter`: 创建路由实例
- `createWebHashHistory`: 使用 hash 模式（URL 带 `#`），适合静态部署
- `redirect`: 路径重定向
- 路由懒加载可以用 `() => import('../views/xxx.vue')`，但这里直接导入

### 1.3 在主文件中引入路由
**文件**: `src/main.js`

```javascript
import { createApp } from "vue";
import App from "./App.vue";
import router from "./router";  // 引入路由

const app = createApp(App);
app.use(router);  // 使用路由插件
app.mount("#app");
```

### 1.4 在模板中使用路由
**文件**: `src/App.vue`

```vue
<!-- 导航链接 -->
<router-link to="/today" class="gh-sidebar-item" active-class="active">
  今日
</router-link>

<!-- 路由出口 -->
<main>
  <router-view />
</main>
```

**知识点**:
- `<router-link>`: 路由链接组件，渲染为 `<a>` 标签但无刷新跳转
- `to`: 目标路径
- `active-class`: 当前激活路由时自动添加的类名
- `<router-view>`: 路由出口，匹配到的组件会渲染在这里

## 二、响应式数据与生命周期

### 2.1 ref 响应式引用
```javascript
import { ref } from 'vue'

const isSidebarOpen = ref(true)  // 创建一个响应式布尔值
```

**知识点**:
- `ref` 包装的值需要通过 `.value` 访问和修改
- 模板中会自动解包，不需要写 `.value`

### 2.2 生命周期钩子
```javascript
import { onMounted, onUnmounted } from 'vue'

onMounted(() => {
  checkScreenWidth()
  window.addEventListener('resize', checkScreenWidth)
})

onUnmounted(() => {
  window.removeEventListener('resize', checkScreenWidth)
})
```

**知识点**:
- `onMounted`: 组件挂载后执行，可以操作 DOM 或绑定事件
- `onUnmounted`: 组件卸载前执行，用于清理工作（如移除事件监听）
- 防止内存泄漏：组件销毁时必须清理全局事件监听

### 2.3 窗口大小监听
```javascript
const checkScreenWidth = () => {
  isSidebarOpen.value = window.innerWidth > window.innerHeight
}
```

**知识点**:
- `window.innerWidth/innerHeight`: 获取视口宽高
- 通过比较宽高判断屏幕方向（横屏/竖屏）
- 比固定像素值（如 520px）更适合高 DPI 屏幕

## 三、动态类绑定与样式

### 3.1 对象语法绑定类
```vue
<aside class="gh-sidebar" :class="{ collapsed: !isSidebarOpen }">
```

**知识点**:
- `:class` 是 `v-bind:class` 的简写
- 对象语法：`{ 类名: 布尔值 }`，为 true 时添加该类
- 可以和普通 `class` 共存，最终类名会合并

### 3.2 动态样式绑定
```vue
<span :style="{ transform: isSidebarOpen ? 'rotate(0deg)' : 'rotate(180deg)' }">
```

**知识点**:
- `:style` 绑定对象或数组
- 属性名可以用驼峰或短横线（短横线需加引号）
- 常用于需要计算值的场景

## 四、CSS 过渡与响应式布局

### 4.1 transition 属性
```css
.gh-sidebar {
  transition: all 0.4s ease-in-out;
}
```

**知识点**:
- `transition`: 简写属性，包含 property duration timing-function delay
- `all`: 所有可过渡属性都参与动画
- `ease-in-out`: 缓动函数，开始和结束慢，中间快
- 必须有过渡前后的属性变化才能看到效果

### 4.2 相邻兄弟选择器
```css
.gh-sidebar.collapsed + .sidebar-toggle {
  left: 16px;
}
```

**知识点**:
- `A + B`: 选择紧跟在 A 后面的 B 元素
- 这里用于根据侧边栏状态控制按钮位置
- 要求元素在 DOM 中相邻

### 4.3 媒体查询 orientation
```css
@media (orientation: portrait) {
  /* 竖屏样式 */
}

@media (orientation: landscape) {
  /* 横屏样式 */
}
```

**知识点**:
- `orientation: portrait`: 竖屏（高 > 宽）
- `orientation: landscape`: 横屏（宽 > 高）
- 比固定宽度媒体查询更适合移动端适配

### 4.4 calc() 函数
```css
.sidebar-toggle {
  left: calc(100% - 46px);
}
```

**知识点**:
- `calc()`: 计算 CSS 属性值
- 可以混合使用不同单位（如 % 和 px）
- 运算符两侧需要空格

### 4.5 绝对定位与 z-index
```css
.gh-sidebar {
  position: absolute;
  top: 0;
  left: 0;
  height: 100%;
  z-index: 100;
}
```

**知识点**:
- `position: absolute`: 相对于最近的定位祖先元素定位
- `z-index`: 控制层叠顺序，值大的在上面
- 需要配合 `position` 不为 static 才有效

### 4.6 flex-shrink
```css
.gh-sidebar {
  flex-shrink: 0;  /* 不收缩 */
}
```

**知识点**:
- `flex-shrink: 0`: 防止 flex 子项在空间不足时收缩
- 侧边栏需要固定宽度，不能被子项压缩

### 4.7 min-width 与 width 配合
```css
.gh-sidebar {
  width: 260px;
  min-width: 260px;
  transition: all 0.4s ease-in-out;
}

.gh-sidebar.collapsed {
  width: 0px;
  min-width: 0px;
}
```

**知识点**:
- 同时设置 `width` 和 `min-width` 确保过渡动画正常
- 某些浏览器下只改 `width` 可能不触发过渡

## 五、事件处理

### 5.1 点击事件
```vue
<button @click="toggleSidebar">
```

**知识点**:
- `@click` 是 `v-on:click` 的简写
- 可以直接写方法名，也可以写内联表达式

### 5.2 方法定义
```javascript
const toggleSidebar = () => {
  isSidebarOpen.value = !isSidebarOpen.value
}
```

**知识点**:
- 箭头函数简洁写法
- 切换布尔值的惯用写法：`!value`

## 六、遇到的问题与解决方案

### 6.1 侧边栏内容在收缩时溢出
**问题**: 宽度变为 0 时文字被挤出来
**解决**: 添加 `overflow: hidden` 和 `opacity: 0`

### 6.2 按钮位置过渡不流畅
**问题**: 使用 `left` 和 `right` 混合控制导致瞬移
**解决**: 统一使用 `left` 属性，用 `calc(100% - 46px)` 计算右侧位置

### 6.3 小屏幕下按钮被侧边栏盖住
**问题**: 展开时按钮消失
**解决**: 提高按钮 z-index 到 101，高于侧边栏的 100
